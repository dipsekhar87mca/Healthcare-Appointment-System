# Sprint Objective

Deliver the foundational capabilities required for users to securely access the Healthcare Appointment System and prepare the platform for appointment management functionality in future sprints.

---

# Sprint Scope

## Features

* User Authentication
* Patient Registration
* User Login
* Role-Based Access Control

---

# Acceptance Criteria

The sprint will be considered successful when:

* Users can register successfully.
* Registered users can log in securely.
* Password validation is enforced.
* User profiles are created after registration.
* Appropriate access is provided based on user role.

---

# Definition of Done

Each completed User Story has:

* Met all acceptance criteria.
* Passed functional testing.
* Passed code review.
* Been successfully integrated.
* Been accepted by the Product Owner.
