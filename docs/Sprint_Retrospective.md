# Retrospective Goal

Reflect on Sprint 1, celebrate successes, identify improvement opportunities, and agree on actionable steps to enhance team effectiveness in future sprints.

---

# Sprint Summary

Sprint 1 focused on delivering the application's foundational capabilities, including patient registration, secure login, user profile creation, and role-based authentication.

The Sprint Goal was successfully achieved, and all committed User Stories met the Definition of Done.

---

# What Went Well ✅

* Sprint Goal achieved successfully.
* Daily Scrum meetings were effective and time-boxed.
* Team collaboration remained strong throughout the sprint.
* Product Owner was available for requirement clarification.
* Acceptance Criteria were clearly defined before development.
* Code reviews were completed on schedule.
* No Critical or High severity production defects were identified.
* Sprint Review received positive stakeholder feedback.

---

# What Could Be Improved ⚠️

* User Stories should be refined earlier to reduce Sprint Planning discussions.
* Test data preparation took longer than expected.
* API documentation should be completed earlier in the sprint.
* Cross-functional knowledge sharing can be improved.
* Environment readiness should be validated before sprint execution.

---

# Root Cause Analysis

| Observation                        | Root Cause                                            |
| ---------------------------------- | ----------------------------------------------------- |
| Test environment delays            | Infrastructure setup was completed later than planned |
| Longer Sprint Planning discussions | A few stories required additional clarification       |
| Delayed API documentation          | Documentation was started too late in the sprint      |

---

# Action Items

| Action                                                     | Owner            | Target Sprint |
| ---------------------------------------------------------- | ---------------- | ------------- |
| Conduct backlog refinement one week before Sprint Planning | Product Owner    | Sprint 2      |
| Prepare test environment before sprint starts              | DevOps Engineer  | Sprint 2      |
| Complete API documentation during development              | Development Team | Sprint 2      |
| Schedule knowledge-sharing sessions every sprint           | Scrum Master     | Sprint 2      |
| Review Definition of Ready before Sprint Planning          | Scrum Team       | Every Sprint  |
